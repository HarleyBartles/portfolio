# Final edit

Use this short pass after drafting or after a targeted revision. Stop when the
pass finds no material issue; do not keep rewriting merely to demonstrate work.

## Bounded checklist

- [ ] The prose answers the reader's actual question or supports the intended
      action.
- [ ] The meaning, qualifications, names, numbers, and uncertainty are intact.
- [ ] The first sentence or heading gives the reader useful orientation.
- [ ] Each paragraph and list has one clear job.
- [ ] Important actors, actions, dates, and consequences are concrete.
- [ ] Unnecessary passive voice, hedging, repetition, jargon, and filler are
      gone or justified.
- [ ] Sentence joins, modifiers, punctuation, and list grammar are sound.
- [ ] Tone matches the audience and does not become promotional, vague, or
      falsely certain.
- [ ] Markdown, UI, error, link, and code formatting follow local conventions.
- [ ] The result still sounds like the author and remains easy to scan.

If a checklist item exposes a specific defect, return to the corresponding
short reference instead of loading the entire reference library.

**Source basis:** The historical source, Rules 8-18 as a review lens,
supplemented by first-party audience, accuracy, and formatting guidance. The
historical source is context for edge cases, not default operational guidance.
