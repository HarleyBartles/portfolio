# Design baseline

## Read when

Read for repo-backed design, discovery, or specification work before invoking
the selected design lane.

## Baseline

Establish the observable problem, affected source and projection seams,
repository doctrine, and local design guide. Separate durable source evidence
from issue or report context. State non-goals, decisions requiring human
authority, expected validation, and the handoff artifact for planning.

Do not turn design into implementation, a local runbook, or a replacement for
brainstorming. The local guide supplies repository paths and constraints; the
design lane supplies design technique. Preserve authored-skill custody and
runtime separation while considering source, generated, install, and
publication surfaces.
