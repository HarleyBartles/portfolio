# Code review baseline

## Read when

Read for repo-backed review, review response, closeout, or merge-readiness
before invoking the selected review lane.

## Baseline

Review the requested diff against the approved plan, repository doctrine,
local code-review guide, source-custody boundary, generated-surface proof, and
validation evidence. Findings identify the requirement, affected surface, and
severity; resolve critical and important findings before closeout or record the
lawful owner of a deferred item.

Do not replace requesting-code-review with a prose checklist or accept local
tests as publication proof. Review confirms that thin routers stayed thin,
README and generated-index boundaries held, cleanup preserved custody, and the
published PR exposes the final diff and checks.
