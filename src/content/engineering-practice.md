# Engineering Practice

Harley's engineering practice starts with requirements and constraints. The goal is not to apply a favorite architecture everywhere; it is to understand the problem well enough to choose the simplest shape that still protects the important behavior.

Repository design is part of that practice. Root guidance should be always available, scoped routers should point to local rules, and durable doctrine should live where future workers can find it. Tests, scripts, worktrees, generated indexes, and review reports are not decoration. They are feedback loops that make work repeatable.

Agent-oriented engineering raises the bar rather than lowering it. Agents can produce useful implementation leverage, but the engineer still owns the requirements, trade-offs, validation, review, and responsibility for the result.
