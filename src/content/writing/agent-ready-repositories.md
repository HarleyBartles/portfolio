# Agent-Ready Repositories

Agent-ready repositories are not repositories with enormous prompts pasted everywhere. They are repositories with progressive discovery: start from a small always-on router, then lead workers to the exact local guidance, policy, tests, and scripts that matter for the current task.

The pattern reduces noise. Root `AGENTS.md` gives the durable entry rules. Scoped `AGENTS.md` files route by directory. Doctrine explains stable decisions. Runbooks describe repeatable operations. Generated `INDEX.md` files help workers navigate without becoming the source of policy.

This matters because agent context is limited and action is cheap. A repository that separates law, navigation, scripts, and source truth makes it easier to keep work narrow, verify claims, and preserve public/private boundaries.
